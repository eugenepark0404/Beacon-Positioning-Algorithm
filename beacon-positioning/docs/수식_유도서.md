## 1. RSSI → 거리: 로그-거리 경로손실 모델 (P1 Eq.(1), P3 Eq.(1)–(3))

### 1.1 원문 수식

P1 Eq.(1):

```
RSSI(d) = RSSI(d0) − 10·n·log10(d/d0)
```

P3 Eq.(1) (경로손실 형태):

```
PL(d)[dB] = PL(d0)[dB] + 10·n·log10(d/d0)
```

### 1.2 기호 정의

| 기호 | 의미 | 문헌 값 |
|------|------|---------|
| `RSSI(d0)` = A | 기준거리 d0(보통 1 m)에서의 평균 RSSI. iBeacon "Measured Power"/Eddystone Tx Power 필드에 해당 | P3 실측: −54.0 ~ −57.7 dBm (비콘별 상이, σ=1.49~2.55 dB) |
| `n` | 경로손실 지수 | 자유공간 2.0. 실내 통상 2.0~3.5 (P3 §III-C). P3 실험 최적 1.8~2.4. 도파관(복도)에서 <2 가능 (P1 §1) |
| `d0` | 기준 거리 | 1 m |
| `d` | 송수신기 간 거리 [m] | — |

### 1.3 역변환 유도 (검산 포함)

RSSI = A − 10n·log10(d/d0) 에서:

```
10n·log10(d/d0) = A − RSSI
log10(d/d0) = (A − RSSI) / (10n)
d = d0 · 10^((A − RSSI)/(10n))
```

**부호 검산**: d = d0(1 m), n=2, A=−59 → RSSI = −59 − 0 = −59 ✓.
d = 10 m → RSSI = −59 − 20·log10(10) = −79 dBm ✓ (멀수록 약함).
역변환: RSSI=−79 → d = 10^((−59−(−79))/20) = 10^1 = 10 m ✓.

> **주의**: P3 본문 Eq.(2)–(3)은 PL→RSSI 치환 시 부호가 +10n으로 인쇄되어 있으나,
> 같은 논문의 실측 추세선(RSSI = −6.338·ln d − 66.765, LOS)과 모순되므로
> 물리적으로 올바른 위 형태(P1 Eq.(1)과 동일)를 채택한다. — 코드 `distance/path_loss.py`

### 1.4 캘리브레이션 (트랙3 주입 지점)

- n, A는 **비콘·채널별로 실측 회귀**로 구한다 (P1 §2: 채널별 empirical path loss exponent 권장).
- 회귀: 거리별 RSSI 측정 → x=10·log10(d/d0), y=RSSI 에 대한 선형최소제곱 → 기울기=−n, 절편=A.
- P4 대안 모델: `d = α + β(R/T) + γ` (신호비 회귀, P4 Eq.(1)–(2)) — 옵션 훅으로만 구현.
- P4 결론: RSSI–거리 변환 최적 구간 0.6~1 m, 원거리일수록 오차 증가 → 가중치 설계 근거.

## 2. 위치 추정

### 2.1 선형화 최소제곱 다변측량 (P4 Eq.(3)–(5))

비콘 i 좌표 Sᵢ=[Sᵢ,₁, Sᵢ,₂], 측정거리 rᵢ에 대해 ‖Sᵢ−P‖² = rᵢ².
1번째 식에서 i번째 식을 빼면 X², Y² 항이 소거되어 선형계가 된다 (P4 Eq.(5)):

```
b = M · [X, Y]^T
Mᵢ = [−2S₁,₁+2Sᵢ,₁ , −2S₁,₂+2Sᵢ,₂]          (i = 2..N)
bᵢ = (r₁²−S₁,₁²−S₁,₂²) − (rᵢ²−Sᵢ,₁²−Sᵢ,₂²)
```

최소제곱해: `[X,Y] = argmin ‖M·p − b‖²` (numpy lstsq).
N=3이면 삼변측량, N>3이면 다변측량 (P4 §2.3.2).

**검산**: 비콘 (0,0),(10,0),(0,10), 실제 위치 (3,4) → r=(5, √65, √45).
b₂ = (25−0) − (65−100) = 60, M₂=[20,0] → X=3 ✓. b₃ = 25 − (45−100) = 80, M₃=[0,20] → Y=4 ✓.

### 2.2 비선형 최소제곱 (P1 §2)

P1은 보정된 RSSI로 구한 채널별 거리를 **비선형 파라메트릭 최소제곱**으로 푼다:

```
p* = argmin_p Σᵢ wᵢ·(‖Sᵢ − p‖ − rᵢ)²
```

scipy.optimize.least_squares (초기값 = 선형해 또는 가중중심). 가중 wᵢ=1/rᵢ^g — 근거리 신뢰(P4 §3.1).

### 2.3 가중 중심 WCL (P3 Eq.(6)–(7), Subedi 2016 재인용)

```
(x,y) = Σᵢ (xᵢ,yᵢ)·wᵢ / Σᵢ wᵢ ,   wᵢ = 1/dᵢ^g
```

g(가중지수): P3 실험 최적 **2.0~3.5** (0.5는 성능 최악). 원이 교차하지 않는 삼변측량 실패 시 fallback (P3 §III-D.1 case 3).

### 2.4 확률 기반 그리드 측위 (P3 Eq.(8)–(9), Knauth 2014 재인용; P5 GML 유사)

```
p(d, dᵢ) = 1 / ((d − dᵢ)² + c)
p(xⱼ,yⱼ) = Πᵢ p(‖(xᵢ,yᵢ)−(xⱼ,yⱼ)‖, dᵢ)   → argmax 그리드 탐색
```

c: 첨예도 (P3 탐색 0.5~3.5). P5(FP-BP)의 grid-based ML(GML)도 동일 계열 → 트랙2 융합 전 BLE 단독 추정기로 채택.

### 2.5 핑거프린팅 + KNN/WKNN (P8 근거, 표준 기법)

P8 리뷰에서 KNN 계열이 54건으로 최다 사용 → 베이스라인 채택.
신호공간 거리: `Dⱼ = ‖f_query − f_j‖₂` (RSSI 벡터, 결측=매우 약한 값으로 대치).
KNN: 상위 k개 참조점 좌표 평균. WKNN: `wⱼ = 1/(Dⱼ+ε)` 가중 평균.
(표준 수식 — P8이 통계적 근거, 세부 변형은 P8 본문 미정독이므로 표준형만 구현)

### 2.6 GML — 그리드 최대우도 측위 (P5 §III, Eq.(9)–(22)) [2차 정독으로 추가]

**핵심 통찰**: RSSI 노이즈 X~N(0,σ̂²)가 로그-거리 모델 `R = −10n·log d + R₀ + X` (P5 Eq.(1),
P1 Eq.(1)과 동일)를 거치면, 추정거리는 **로그정규분포**를 따른다:

```
log d̂ᵢ ~ N(log dᵢ, ηᵢ²),   ηᵢ = σ̂ᵢ/(10n)          (P5 Eq.(11)–(12))
E[d̂ᵢ] = dᵢ·exp[½(σ̂ᵢ·ln10/10n)²] > dᵢ              (P5 Eq.(16)) ← 거리 과대추정 편향!
```

즉 d̂를 그대로 참거리로 쓰면 체계적 편향이 생기므로, 우도를 **로그 거리 도메인**에서 세운다:

```
y* = argmin_{y∈G'} Σᵢ (1/2ηᵢ²)·(log d̂ᵢ − log‖y−bᵢ‖)²      (P5 Eq.(19))
```

- 비콘 선택: RSSI 상위 N≥3개 (P5 Eq.(9))
- 후보 격자 G': 전체 격자 ∩ 선택 비콘 볼록껍질 내부, 이전 위치와 맨해튼 거리 < d₀
  (P5 Eq.(20); 희소 배치면 껍질 제약 제외, Eq.(21))
- 시간 평활: 최근 n개 결과 평균 (P5 Eq.(22))
- P5 파라미터: 격자 0.3 m, PF 파티클 500, 광고 200 ms
- P5 실측: 보행 중 GML 평균 2.01 m — 삼변측량 9.39 m, 일반 ML 2.63 m 대비 우수.
  FP-BP(GML+PDR+PF+도면보정) 1.19 m.

**검산**: 무노이즈면 d̂ᵢ=dᵢ → 참 위치에서 비용 0 = 전역 최소 ✓. E[d̂] 편향식은
로그정규 평균 공식 e^{μ+σ²/2}의 밑-10 변환과 일치 ✓.
구현: `positioning/gml.py` (tests/test_positioning.py 에서 NLS 대비 로그정규 노이즈 하 우위 검증).

### 2.7 퍼지 가중 핑거프린팅 (P9 §2.2, Eq.(1)–(7)) [2차 정독으로 추가]

표준 신호공간 거리(P9 Eq.(2))의 문제: 대칭 위치에서 같은 값 → k>1일 때 오차 증가 (P9 §2.2.2).
해결: 비콘 j별 퍼지 가중 w를 넣은 가중 유클리드 거리 (마할라노비스 유사):

```
Dᵢ = √( Σⱼ ((Pᵢⱼ − P'ⱼ)·wᵢⱼ)² )        (P9 Eq.(5))
좌표 = Σ wᵢ(Xᵢ,Yᵢ)/Σ wᵢ  (WKNN)        (P9 Eq.(4))
```

- wᵢⱼ = FIS(기하거리, RSSI): 삼각 멤버십, Mamdani max-min, 무게중심 역퍼지화
- 도메인 (P9 §2.2.2 명시): 거리 0~10 m, RSSI −90~−20 dBm, 가중 0~1
- Type-2 = hesitant fuzzy set: 상/하 삼각 MF 두 벌 → 구간 축약 (P9 Eq.(6)–(7))
- P9 결과: 평균 0.43 m, 90%가 0.63 m 이내, 내비 정밀도 98.2±1%
- **한계 명시**: MF 절점·규칙표 수치는 P9 그림/부록(이미지)이라 추출 불가.
  P9 스스로 "휴리스틱하게 선택"(§2.2.2)이라 밝혔으므로, 구현(`positioning/fuzzy.py`)의
  절점도 도메인 정의만 P9를 따르고 값은 config 인자로 교체 가능한 휴리스틱이다.

## 3. 인체 차폐(body-shadowing) 감지·보정 — 이 과제의 심장 (P1)

### 3.1 물리적 근거 (P1 §1–2)

- BLE 광고는 3개 채널 사용: **37(2402 MHz), 38(2426 MHz), 39(2480 MHz)**.
- **다중경로 페이딩은 채널별로 다르게** 나타난다 (주파수 의존).
- **인체 차폐는 3개 채널을 동시에** 감쇠시킨다.
- → "3채널 동시 하락 = 차폐, 일부 채널만 요동 = 다중경로"가 판별 원리.
- 차폐 감쇠량은 인체가 수신기에 가까울수록 큼 (P1 §2.2, 6 m 실험).

### 3.2 임계값 기반 감지 (P1 §1의 선행연구 [9] 방식)

각 채널 c의 롤링 평균 μ_c, 표준편차 σ_c에 대해:

```
감지 = ∀c ∈ {37,38,39}:  RSSI_c(t) < μ_c − k·σ_c    (k = 1.0, P1 [9])
```

모든 채널이 동시에 k·σ 이상 하락하면 차폐 이벤트 선언. 경량·해석가능 → 1차 감지기로 구현.

### 3.3 ANN 기반 감지+보정 (P1 §2.3, 본 논문의 핵심 기여)

- **입력**: 채널별 과거 N=10 샘플 슬라이딩 윈도우 (3채널 × 10 = 30차원). N=10 선정 근거: 전이(차폐 시작/종료) 패턴을 담을 최소 길이 (P1 §2.3).
- **출력**: 채널별 보정 RSSI 3개 + 차폐 상태 플래그 1개.
- **구조**: MLP 은닉 2층 × 20뉴런 (P1 §2.3.1, 20뉴런 초과·2층 초과 시 개선 없음). RBF는 은닉 1층 × 20뉴런.
- **학습 타깃**: 같은 수신 위치에서 **차폐 없는** 측정값 (경로손실 모델 값이 아니라 실측 LoS 분산을 보존하기 위함, P1 §2.3).
- 비용함수: `C = (1/2m)Σ‖y−ŷ‖²` (표준 MSE 역전파).
- **성능** (P1 §3): MLP 차폐 감지율 89% (미탐 11%), LoS 정분류 94% (오경보 6%). RBF는 미탐 13%/오경보 8%. 보정 후 삼변측량 오차 CDF 유의미 개선.
- 학습 데이터 분할: 70% 학습 / 30% 시험 (P1 §3 case 2).

### 3.4 우리 프로젝트 적용

1. `correction/shadowing.py`의 `ThresholdShadowDetector` — §3.2 규칙 (실시간·무학습).
2. `MlShadowCorrector` — §3.3 구조 그대로 (scikit-learn MLP, 은닉 (20,20)), 학습/추론 인터페이스 분리.
3. 보정 실패·미학습 시 fallback: 감지된 차폐 구간에 **고정 감쇠 보상** `RSSI += Δ_shadow`.
   Δ_shadow placeholder = 7 dB — 근거: P3 Table II의 LOS/NLOS 실측 추세선 차이
   (LOS: −6.338·ln d − 66.765, NLOS: −3.851·ln d − 75.869 → d=1~4 m에서 약 6~9 dB 차).
   **트랙3 실측값으로 교체 필수** (config placeholder).

### 3.5 다중경로/NLoS 판별 (P1 원리 + P2 §2.2)

P2가 정리한 오차원: 다중경로(반사 합성), Signal Fading(LoS 소실), 노이즈(간섭).
판별 규칙 (P1 3채널 원리의 대우):

```
채널 간 편차 급증 + 전 채널 동시 하락 아님 → 다중경로/페이딩 플래그 (해당 채널 신뢰도 하향)
전 채널 동시 하락                         → 인체 차폐 (3.2/3.3으로 보정)
```

채널 정보가 없는 데이터(집계 RSSI)는 분산 급증 기반 휴리스틱으로 강등 동작.

## 4. RSSI 안정화 필터

### 4.1 슬라이딩 윈도우 (P3 §III-B, §V-A)

- 비콘별 독립 윈도우, 크기 n. P3 실험: **median이 최적** (mean은 유의미하게 나쁨), **윈도우 10** 최적 (1은 최악, 15·20도 양호하나 지연↑).
- 보행속도 5 km/h 기준의 결과 → 윈도우는 이동속도와 트레이드오프 (P3 §V-A.2).

### 4.2 1차원 칼만 필터 (P4 §2.3.4 적용 근거, 표준 스칼라 KF)

RSSI를 상태로 하는 정적 모델 (P4가 KF를 RSSI 안정화에 사용; 수식은 표준형):

```
예측: x̂⁻ = x̂,  P⁻ = P + Q
갱신: K = P⁻/(P⁻+R),  x̂ = x̂⁻ + K(z − x̂⁻),  P = (1−K)P⁻
```

Q(공정잡음)·R(측정잡음) placeholder: R = 관측 분산(P3 실측 σ≈1.5~2.6 dB → R≈4~6), Q는 실측 튜닝 (config).

### 4.3 이상치 제거 (표준 MAD 규칙)

`|RSSI − median| > k·1.4826·MAD → 제거` (k=3). P4 §3.1의 "중앙값 사용" 권고와 부합.

## 5. BLE+IMU 융합 (P2 — 트랙2 연동 규격의 근거)

### 5.1 파티클 필터 골격 (P2 §3.1, Eq.(7)–(8))

```
1. 예측: xᵢ ← f(xᵢ, Δt)                        (Eq.7)
2. 갱신: wᵢ ← wᵢ·P(M|xᵢ)                       (Eq.8)
3. 재샘플: systematic resampling                (P2 [42])
```

### 5.2 이동 모델 (P2 Eq.(9)–(14))

상태 = (X, Y, Vx, Vy). 선형 이동 + 가우시안 요동:

```
X(t+Δt) = X + Vx·Δt + r_xy·Δt,   r_xy ~ N(0, σ_xy²)      (9)
Y(t+Δt) = Y + Vy·Δt + r_xy·Δt                              (10)
Vx(t+Δt) = Vx + r_V·Δt,          r_V ~ N(0, σ_V²)          (11,12)
회전: θ = Δα + r_α, r_α ~ N(0, σ_α²)
Vx' = Vx·cosθ − Vy·sinθ ;  Vy' = Vy·cosθ + Vx·sinθ         (13,14)
```

P2 실험 파라미터 (Eq.15–17): **σ_xy=0.5, σ_V=1.0, σ_α=0.3** → config 기본값.

### 5.3 IMU 우도 (P2 Eq.(18)–(21))

PDR 보속: `V_I(tᵢ) = L_s/(tᵢ−tᵢ₋₁)` (L_s=보폭, Eq.18). Madgwick 필터로 자세 정렬 (P2 §3.3, 자력계는 실내 왜곡으로 비신뢰).

```
L_IMU(Vx,Vy|V_I) = b_V + (1−b_V)·N(Vx²+Vy² | V_I², σ_V²)   (19)
```

P2 값: **b_V=0.3, σ_V=0.7** (Eq.20–21). 베이스라인 b_V는 IMU 노이즈·드리프트 대비.

### 5.4 BLE 우도 (P2 Eq.(22)–(27)) — MCPD 전용부

MCPD(iDFT 스펙트럼→거리 확률분포, Eq.22; 가우시안 가중 Eq.23: b_W=0.1, σ_W=1.0; 정규화 Eq.26; L_BLE Eq.27)는 **채널 사운딩 하드웨어 필요** → 스마트폰 RSSI만으로는 불가.
우리 구현: 동일한 "베이스라인 있는 가우시안 우도" **형태만 차용**해 RSSI 거리/위치 추정치에 적용:

```
L_BLE(X,Y | p_ble) = b_W + (1−b_W)·N(‖(X,Y)−p_ble‖ | 0, σ_ble²)
```

σ_ble = BLE 단독 측위의 실측 오차 표준편차 (placeholder 2.0 m ≈ P3 중앙값 오차대).

### 5.5 출력 (P2 Eq.(28))

`추정치 = Σᵢ wᵢ·stateᵢ` (가중 평균). 파티클 수: P2는 30,000 사용, **10,000으로 정확도 손실 <1%** (P2 §5.6) → 기본 10,000.

### 5.6 성과 근거 (P2 Table 1–2)

Closest peak 68% → PF 77% → **PF+IMU 93%** (±1 m 이내 비율, Setup A). NLoS에서 62→82%, 신호 손실에서 63→95%. → BLE+IMU 융합이 트랙2 핵심 과제임을 뒷받침.
P5(FP-BP)도 PF 융합 + 도면 사후보정으로 평균 1.14 m 달성 → 도면 제약(floor plan constraint) 훅을 인터페이스에 포함.
